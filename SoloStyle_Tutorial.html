1. Введение и обзор проекта

1.1 Что такое SoloStyle?

SoloStyle — это iOS-приложение для фрилансеров в сфере красоты (парикмахеры, визажисты, мастера маникюра и т.д.). Оно позволяет:

  • Управлять клиентской базой
  • Вести календарь записей
  • Отслеживать доходы
  • Создавать и редактировать услуги
  • Использовать AI-ассистента для бизнес-советов

2. Архитектура приложения

2.1 Точка входа (App)

Файл SoloStyleApp.swift — это точка входа. Здесь создаётся контейнер данных и определяется, какой экран показать:

import SwiftUI
import SwiftData

@main
struct SoloStyleApp: App {
    @AppStorage("hasCompletedOnboarding") private var hasCompletedOnboarding = false

    var sharedModelContainer: ModelContainer = {
        let schema = Schema([
            Master.self,
            Client.self,
            Service.self,
            Appointment.self
        ])
        let config = ModelConfiguration(schema: schema, isStoredInMemoryOnly: false)

        do {
            return try ModelContainer(for: schema, configurations: [config])
        } catch {
            // Fallback на in-memory если ошибка
            let memoryConfig = ModelConfiguration(schema: schema, isStoredInMemoryOnly: true)
            return try! ModelContainer(for: schema, configurations: [memoryConfig])
        }
    }()

    var body: some Scene {
        WindowGroup {
            if hasCompletedOnboarding {
                MainTabView()
            } else {
                OnboardingView()
            }
        }
        .modelContainer(sharedModelContainer)
    }
}
Что здесь происходит:
  • @main — указывает, что это точка входа приложения
  • @AppStorage — сохраняет значение в UserDefaults
  • ModelContainer — контейнер для SwiftData (аналог Core Data)
  • Schema — список моделей, которые будут храниться в базе

3. Модели данных (SwiftData)

3.1 Что такое SwiftData?

SwiftData — это современный фреймворк Apple для хранения данных. Это замена Core Data с более простым синтаксисом. Данные хранятся в SQLite базе на устройстве.

3.2 Модель Master (Мастер)

Хранит информацию о владельце приложения:

@Model
final class Master {
    var id: UUID
    var name: String
    var businessName: String?
    var avatarData: Data?
    var publicSlug: String
    var createdAt: Date

    init(name: String, businessName: String? = nil) {
        self.id = UUID()
        self.name = name
        self.businessName = businessName
        self.publicSlug = UUID().uuidString.prefix(8).lowercased()
        self.createdAt = Date()
    }
}
@Model — макрос SwiftData, который превращает класс в модель данных. Все свойства автоматически сохраняются в базу.

3.3 Модель Client (Клиент)

@Model
final class Client {
    var id: UUID
    var name: String
    var phone: String?
    var email: String?
    var notes: String?
    var createdAt: Date

    // Связь с записями (один ко многим)
    @Relationship(deleteRule: .cascade, inverse: \Appointment.client)
    var appointments: [Appointment]

    init(name: String, phone: String? = nil, email: String? = nil) {
        self.id = UUID()
        self.name = name
        self.phone = phone
        self.email = email
        self.createdAt = Date()
        self.appointments = []
    }
}
@Relationship — определяет связь между моделями. deleteRule: .cascade означает, что при удалении клиента удалятся все его записи.

3.4 Модель Service (Услуга)

@Model
final class Service {
    var id: UUID
    var name: String
    var price: Decimal
    var duration: Int  // в минутах
    var isActive: Bool
    var createdAt: Date

    // Вычисляемые свойства для отображения
    var formattedPrice: String {
        return "$\(price)"
    }

    var formattedDuration: String {
        if duration >= 60 {
            let hours = duration / 60
            let mins = duration % 60
            return mins > 0 ? "\(hours)h \(mins)m" : "\(hours)h"
        }
        return "\(duration) min"
    }
}

4.2 Зачем нужна дизайн-система?

  • Единообразие — все элементы выглядят одинаково
  • Быстрые изменения — меняешь в одном месте, меняется везде
  • Читаемость кода — Design.Spacing.m понятнее чем 16
  • Темы — легко добавить тёмную тему

5.1 GlassCard — базовая карточка

Карточка со стеклянным эффектом (Liquid Glass iOS 26):

struct GlassCard<Content: View>: View {
    let tint: Color
    let padding: CGFloat
    @ViewBuilder let content: () -> Content

    init(
        tint: Color = Design.Colors.glassTint,
        padding: CGFloat = Design.Spacing.m,
        @ViewBuilder content: @escaping () -> Content
    ) {
        self.tint = tint
        self.padding = padding
        self.content = content
    }

    var body: some View {
        content()
            .padding(padding)
            .background(
                RoundedRectangle(cornerRadius: Design.Radius.l)
                    .fill(Design.Colors.backgroundSecondary.opacity(0.5))
            )
            .glassEffect(.regular.tint(tint), in: .rect(cornerRadius: Design.Radius.l))
    }
}
.glassEffect() — модификатор iOS 26, который добавляет эффект матового стекла с размытием фона.

5.4 HapticManager — тактильная обратная связь

enum HapticManager {
    static func impact(_ style: UIImpactFeedbackGenerator.FeedbackStyle) {
        let generator = UIImpactFeedbackGenerator(style: style)
        generator.impactOccurred()
    }

    static func notification(_ type: UINotificationFeedbackGenerator.FeedbackType) {
        let generator = UINotificationFeedbackGenerator()
        generator.notificationOccurred(type)

    static func selection() {
        let generator = UISelectionFeedbackGenerator()

Применение:

6.1 CalendarView — Календарь

Главный экран с записями на выбранную дату:

struct CalendarView: View {
    @Environment(\.modelContext) private var modelContext
    @Query private var appointments: [Appointment]

    @State private var selectedDate = Date()
    @State private var showingAddAppointment = false

    // Фильтрация записей по выбранной дате
    private var todayAppointments: [Appointment] {
        appointments.filter { appointment in
            Calendar.current.isDate(appointment.date, inSameDayAs: selectedDate)
        }.sorted { $0.date < $1.date }
    }

    var body: some View {
        NavigationStack {
            ZStack {
                Design.Colors.backgroundPrimary.ignoresSafeArea()

                VStack(spacing: 0) {
                    // Полоска с датой
                    DatePicker("Date", selection: $selectedDate, displayedComponents: .date)
                        .datePickerStyle(.graphical)

                    // Список записей
                    ScrollView {
                        LazyVStack(spacing: Design.Spacing.s) {
                            ForEach(todayAppointments) { appointment in
                                AppointmentCard(appointment: appointment)
                            }
                        }
                        .padding(Design.Spacing.m)
                    }
                }

                // FAB кнопка
                VStack {
                    Spacer()
                    HStack {
                        Spacer()
                        GlassFAB(icon: "plus") {
                            showingAddAppointment = true
                        }
                        .padding()
                    }
                }
            }
            .navigationTitle("Calendar")
            .sheet(isPresented: $showingAddAppointment) {
                AddAppointmentView(selectedDate: selectedDate)
            }
        }
    }
}

6.2 ClientsView — Клиенты

Список клиентов с поиском и фильтрацией:

struct ClientsView: View {
    @Environment(\.modelContext) private var modelContext
    @Query(sort: \Client.name) private var clients: [Client]

    @State private var searchText = ""
    @State private var selectedFilter: ClientFilter = .all

    private var filteredClients: [Client] {
        var result = clients

        // Поиск
        if !searchText.isEmpty {
            result = result.filter {
                $0.name.localizedCaseInsensitiveContains(searchText)
            }
        }

        // Фильтры
        switch selectedFilter {
        case .all: break
        case .recent:
            let thirtyDaysAgo = Calendar.current.date(byAdding: .day, value: -30, to: Date())!
            result = result.filter { $0.appointments.contains { $0.date > thirtyDaysAgo } }
        case .frequent:
            result = result.filter { $0.appointments.count >= 3 }
        case .new:
            result = result.filter { $0.appointments.isEmpty }
        }

        return result
    }
}

8. Кэширование и оптимизация

8.1 StatsCache

Для оптимизации запросов к базе используется кэш статистики:

@MainActor
final class StatsCache {
    static let shared = StatsCache()

    private(set) var clientCount: Int = 0
    private(set) var appointmentCount: Int = 0
    private(set) var activeServicesCount: Int = 0
    private(set) var thisMonthAppointments: Int = 0
    private(set) var thisMonthEarnings: Decimal = 0
    private(set) var thisYearEarnings: Decimal = 0

    private var lastRefresh: Date?
    private let cacheTimeout: TimeInterval = 5 // 5 секунд

    func refreshIfNeeded(context: ModelContext) {
        if let last = lastRefresh,
           Date().timeIntervalSince(last) < cacheTimeout {
            return // Кэш ещё свежий
        }
        refresh(context: context)
    }

    func refresh(context: ModelContext) {
        // Подсчёт клиентов
        let clientDescriptor = FetchDescriptor<Client>()
        clientCount = (try? context.fetchCount(clientDescriptor)) ?? 0

        // Подсчёт записей
        let appointmentDescriptor = FetchDescriptor<Appointment>()
        appointmentCount = (try? context.fetchCount(appointmentDescriptor)) ?? 0

        // Заработок за месяц
        let startOfMonth = Calendar.current.date(from:
            Calendar.current.dateComponents([.year, .month], from: Date())
        )!

        let monthDescriptor = FetchDescriptor<Appointment>(
            predicate: #Predicate {
                $0.status == .completed && $0.date >= startOfMonth
            }
        )

        if let monthAppointments = try? context.fetch(monthDescriptor) {
            thisMonthEarnings = monthAppointments.reduce(0) {
                $0 + ($1.service?.price ?? 0)
            }
        }

        lastRefresh = Date()
    }

    func invalidate() {
        lastRefresh = nil
    }
}
Когда вызывать invalidate():
  • После добавления/удаления клиента
  • После создания/удаления записи
  • После изменения статуса записи на completed
  • После добавления/удаления услуги

9. Валидация и безопасность

9.1 InputValidator

enum InputValidator {
    // Проверка email
    static func isValidEmail(_ email: String) -> Bool {
        guard !email.isEmpty else { return true }
        let pattern = "^[A-Z0-9a-z._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,64}$"
        return email.range(of: pattern, options: .regularExpression) != nil
    }

    // Проверка телефона
    static func isValidPhone(_ phone: String) -> Bool {
        guard !phone.isEmpty else { return true }
        let digits = phone.filter { $0.isNumber }
        return digits.count >= 7 && digits.count <= 15
    }

    // Проверка имени
    static func isValidName(_ name: String) -> Bool {
        let trimmed = name.trimmingCharacters(in: .whitespacesAndNewlines)
        return trimmed.count >= 2 && trimmed.count <= 100
    }

    // Проверка цены
    static func isValidPrice(_ price: String) -> Bool {
        guard let value = Decimal(string: price) else { return false }
        return value >= 0 && value <= 1_000_000
    }

    // Очистка от опасных символов
    static func sanitize(_ input: String) -> String {
        var result = input.trimmingCharacters(in: .whitespacesAndNewlines)
        // Удаляем потенциально опасные символы
        let dangerous: [Character] = ["<", ">", "&", "\"", "'"]
        result.removeAll { dangerous.contains($0) }
        return result
    }
}

9.2 Применение валидации

private func saveClient() {
    // Проверяем все поля
    guard InputValidator.isValidName(name) else {
        HapticManager.notification(.error)
        return
    }
    guard InputValidator.isValidPhone(phone) else {
        HapticManager.notification(.error)
        return
    }
    guard InputValidator.isValidEmail(email) else {
        HapticManager.notification(.error)
        return
    }

    // Очищаем данные перед сохранением
    let client = Client(
        name: InputValidator.sanitize(name),
        phone: InputValidator.sanitize(phone),
        email: InputValidator.sanitize(email)
    )
    modelContext.insert(client)
}

  • Master — данные владельца
  • Client — клиенты
  • Service — услуги
  • Appointment — записи

Шаг 3: Дизайн-система

Создай DesignSystem.swift со всеми константами (цвета, отступы, шрифты). Это упростит поддержку.

Шаг 4: Компоненты

Создай переиспользуемые компоненты в GlassComponents.swift:

  • GlassCard
  • GlassButton
  • FormField
  • GlassFAB

Шаг 5: Экраны

Создай основные экраны:

  1. OnboardingView — первый запуск
  2. MainTabView — навигация
  3. CalendarView — календарь
  4. ClientsView — клиенты
  5. ProfileView — профиль
  6. SettingsView — настройки

Шаг 6: Оптимизация

  • Добавь StatsCache для кэширования
  • Добавь индексы в модели
  • Используй LazyVStack вместо VStack для списков

Шаг 7: Безопасность

  • Добавь InputValidator
  • Валидируй все входные данные
  • Очищай данные от опасных символов

Шаг 8: AI (опционально)
